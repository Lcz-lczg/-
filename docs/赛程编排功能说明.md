# 赛程自动编排功能开发文档

## 功能概述

赛程自动编排功能允许管理员根据参赛球队数量和比赛模式，自动生成完整的比赛赛程。系统支持单循环、双循环和淘汰制三种比赛模式，并可选择是否按球队实力水平进行种子编排。

## 技术架构

### 后端技术
- **框架**: Spring Boot 2.3.12
- **数据库访问**: MyBatis-Plus + DB-Query
- **算法实现**: 贝格尔编排法 (Berger pairing system)

### 前端技术
- **框架**: Vue 3 + Element Plus
- **状态管理**: Pinia
- **路由**: Vue Router 4

## 赛程编排算法

### 1. 单循环赛 (Single Round Robin)

**算法**: 贝格尔编排法 (Berger pairing system)

**特点**:
- 每支球队与其他球队各比赛一次
- 比赛场次 = n × (n-1) / 2（n 为球队数量）
- 使用固定轮转法，确保公平性

**编排逻辑**:
```
1. 如果队伍数量是奇数，添加一个虚拟队伍（轮空）
2. 固定第一支队伍，其他队伍顺时针旋转
3. 每轮生成 n/2 场比赛
4. 共进行 n-1 轮比赛
```

**示例**（4 支球队）:
```
第 1 轮：1 vs 4, 2 vs 3
第 2 轮：1 vs 3, 4 vs 2
第 3 轮：1 vs 2, 3 vs 4
```

### 2. 双循环赛 (Double Round Robin)

**特点**:
- 每支球队与其他球队各比赛两次（主客场）
- 比赛场次 = n × (n-1)
- 在单循环基础上生成第二循环（主客场互换）

**编排逻辑**:
```
1. 先生成单循环赛程
2. 复制所有比赛，交换主客场
3. 第二循环标记为" (次)"
```

**示例**（4 支球队）:
```
第一循环:
  第 1 轮：1 vs 4, 2 vs 3
  第 2 轮：1 vs 3, 4 vs 2
  第 3 轮：1 vs 2, 3 vs 4

第二循环:
  第 1 轮 (次): 4 vs 1, 3 vs 2
  第 2 轮 (次): 3 vs 1, 2 vs 4
  第 3 轮 (次): 2 vs 1, 4 vs 3
```

### 3. 淘汰赛 (Knockout)

**特点**:
- 单败淘汰制，输一场即出局
- 参赛队伍数量必须是 2 的幂次（2、4、8、16...）
- 比赛场次 = n - 1
- 强队可设置轮空，避免过早相遇

**编排逻辑**:
```
1. 按实力排序球队（如果启用种子编排）
2. 计算轮空数量（补齐到 2 的幂次）
3. 强队直接晋级（轮空）
4. 其他球队配对进行第一轮
5. 逐轮生成后续比赛（对手待定）
```

**示例**（8 支球队）:
```
第 1 轮（8 进 4）: 
  第 1 场：1 号种子 vs 8 号种子
  第 2 场：4 号种子 vs 5 号种子
  第 3 场：2 号种子 vs 7 号种子
  第 4 场：3 号种子 vs 6 号种子

第 2 轮（半决赛）:
  第 1 场：第 1 场胜者 vs 第 2 场胜者
  第 2 场：第 3 场胜者 vs 第 4 场胜者

第 3 轮（决赛）:
  第 1 场：半决赛第 1 场胜者 vs 半决赛第 2 场胜者
```

## 实力水平编排（种子编排）

**原理**:
- 根据球队的 `shilidengji`（实力等级，1-5 级）进行排序
- 强队（高等级）在淘汰赛中后相遇
- 循环赛中使用贝格尔编排法自动平衡

**种子编排示例**（淘汰赛）:
```
球队实力排序：A(5 级), B(5 级), C(4 级), D(4 级), E(3 级), F(3 级), G(2 级), H(1 级)

编排对阵:
  A(5) vs H(1)  - 最强 vs 最弱
  D(4) vs E(3)
  B(5) vs G(2)  - 次强 vs 次弱
  C(4) vs F(3)

这样确保 A 和 B 两支最强队伍只在决赛相遇
```

## 前端页面流程

### 步骤 1: 选择比赛
- 从下拉列表选择比赛信息

- 显示生成的赛程预览

- 确认后保存到数据库

## 注意事项

### 1. 淘汰赛队伍数量限制
- 必须是 2 的幂次：2、4、8、16、32...
- 如果队伍数量不足，系统会提示
- 解决方案：添加虚拟队伍（轮空）或调整赛制

### 2. 实力等级设置
- 实力等级范围：1-5 级
- 建议在球队信息中提前设置
- 影响种子编排的公平性

### 3. 赛程保存
- 预览后可以批量保存
- 保存后可以在赛程表页面编辑
- 支持清除重新生成

### 4. 比赛结果录入
- 赛程生成后状态为"待开始"
- 比赛完成后在"比赛完成"模块录入结果
- 系统自动更新积分榜

## 常见问题

### Q1: 赛程生成失败？
**A**: 检查以下条件：
- 是否选择了比赛信息
- 参赛球队数量是否≥2
- 淘汰赛队伍数量是否为 2 的幂次

### Q2: 实力编排不起作用？
**A**: 确保：
- 球队的实力等级已设置
- 在步骤 3 中启用了"按实力种子编排"

### Q3: 保存赛程时报错？
**A**: 检查：
- 数据库连接是否正常
- 是否有重复的赛程数据
- 尝试先清除已有赛程
